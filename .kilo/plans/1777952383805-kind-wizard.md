# Kế hoạch: Thêm Custom Prompt vào Settings

## Tổng quan

Thêm một setting "Custom Prompt" vào tab Settings. Custom prompt này sẽ được chèn vào prompt đã tạo **trước** phần `<user_instructions>`. Người dùng có thể chọn phạm vi lưu: **Project** (workspaceState – chỉ áp dụng cho workspace hiện tại) hoặc **Global** (globalState – áp dụng cho tất cả các project).

---

## Luồng hoạt động

1. Người dùng nhập custom prompt vào textarea trong Settings tab.
2. Chọn scope: "Project" hoặc "Global" (radio/toggle).
3. Nhấn Save → nội dung được lưu vào workspaceState (project) hoặc globalState (global).
4. Khi copy context, backend tải custom prompt dựa trên scope preference, rồi chèn vào prompt **trước** `<user_instructions>`.

---

## Thiết kế lưu trữ

- `promptforge.customPromptProject` – workspaceState: nội dung custom prompt cấp project
- `promptforge.customPromptGlobal` – globalState: nội dung custom prompt cấp global
- `promptforge.customPromptScope` – workspaceState: `'project' | 'global'` (scope hiện tại đang dùng; mặc định `'global'`)

Khi load settings để hiển thị trên UI:
- Load cả `customPromptProject` và `customPromptGlobal`
- Load `customPromptScope`
- Hiển thị textarea với nội dung ứng với scope đang chọn

Khi generate prompt (copy context):
- Đọc `customPromptScope`
- Nếu `'project'` → dùng `customPromptProject`
- Nếu `'global'` → dùng `customPromptGlobal`

### 2. `src/providers/file-explorer/index.ts`

**Thêm storage keys:**
```typescript
private static readonly CUSTOM_PROMPT_PROJECT_KEY = 'promptforge.customPromptProject'
private static readonly CUSTOM_PROMPT_SCOPE_KEY = 'promptforge.customPromptScope'
// Global key dùng globalState nên không cần prefix workspace
```

**Cập nhật `_loadSettings()`:**
```typescript
private _loadSettings(): UpdateSettingsPayload {
  // ... existing fields ...
  const customPromptProject = this._context.workspaceState.get<string>(
    FileExplorerWebviewProvider.CUSTOM_PROMPT_PROJECT_KEY, ''
  )
  const customPromptGlobal = this._context.globalState.get<string>(
    'promptforge.customPromptGlobal', ''
  )
  const customPromptScope = this._context.workspaceState.get<'project' | 'global'>(
    FileExplorerWebviewProvider.CUSTOM_PROMPT_SCOPE_KEY, 'global'
  )
  return { excludedFolders, excludedExtensions, readGitignore, customPromptProject, customPromptGlobal, customPromptScope }
}
```

**Cập nhật `_saveSettings()`:**
```typescript
private async _saveSettings(payload: SaveSettingsPayload): Promise<void> {
  // ... existing saves ...
  await this._context.workspaceState.update(
    FileExplorerWebviewProvider.CUSTOM_PROMPT_PROJECT_KEY, payload.customPromptProject
  )
  await this._context.globalState.update(
    'promptforge.customPromptGlobal', payload.customPromptGlobal
  )
  await this._context.workspaceState.update(
    FileExplorerWebviewProvider.CUSTOM_PROMPT_SCOPE_KEY, payload.customPromptScope
  )
}
```

**Cập nhật `_handleCopyContext()` để truyền customPrompt vào `generatePrompt()`:**
```typescript
const settings = this._loadSettings()
const customPrompt = settings.customPromptScope === 'project'
  ? settings.customPromptProject
  : settings.customPromptGlobal

const prompt = generatePrompt(fileMap, fileContents, userInstructions, includeXml, payload.mode, customPrompt)
```

### 3. `src/prompts/index.ts`

**Cập nhật `generatePrompt()` để nhận và chèn customPrompt:**

Thêm param `customPrompt?: string`. Chèn vào **trước** `<user_instructions>`:

```typescript
export function generatePrompt(
  fileMap: string,
  fileContents: string,
  userInstructions: string,
  includeXmlFormatting: boolean,
  mode?: 'plan' | 'code',
  customPrompt?: string,   // NEW
): string {
  let prompt = `<file_map>\n${fileMap}\n</file_map>\n\n<file_contents>\n${fileContents}\n</file_contents>\n`

  if (includeXmlFormatting) {
    prompt += `\n${XML_FORMATTING_INSTRUCTIONS}`
  }

  // NEW: Custom prompt chèn trước user_instructions
  if (customPrompt && customPrompt.trim() !== '') {
    prompt += `\n<custom_instructions>\n${customPrompt.trim()}\n</custom_instructions>\n`
  }

  if (userInstructions && userInstructions.trim() !== '') {
    prompt += `\n<user_instructions>\n${userInstructions.trim()}\n</user_instructions>\n`
  }

  // ... mode instructions ...
  return prompt
}
```

### 4. `src/webview-ui/src/components/settings-tab/custom-prompt.tsx` *(file mới)*

Component mới hiển thị:
- Label "Custom Prompt"
- Textarea để nhập nội dung (resize vertical)
- Scope toggle: hai button "Project" / "Global"
  - Khi đổi scope, textarea hiện nội dung ứng với scope đó
- Mô tả ngắn: "Inserted before User Instruction in every generated prompt"

```tsx
interface CustomPromptProps {
  customPromptProject: string
  customPromptGlobal: string
  customPromptScope: 'project' | 'global'
  onChangeProject: (v: string) => void
  onChangeGlobal: (v: string) => void
  onChangeScope: (v: 'project' | 'global') => void
}
```

Logic:
- Hiển thị textarea với giá trị là `customPromptProject` nếu scope = 'project', hoặc `customPromptGlobal` nếu scope = 'global'
- `onInput` của textarea gọi `onChangeProject` hoặc `onChangeGlobal` tuỳ scope hiện tại
- Scope toggle gọi `onChangeScope`

## Cấu trúc XML output

Khi custom prompt có nội dung, prompt được tạo ra có dạng:

```xml
<file_map>...</file_map>

<file_contents>...</file_contents>

[XML_FORMATTING_INSTRUCTIONS nếu bật]

<custom_instructions>
[nội dung custom prompt]
</custom_instructions>

<user_instructions>
[nội dung user instruction]
</user_instructions>

[mode_instructions nếu có]
```
