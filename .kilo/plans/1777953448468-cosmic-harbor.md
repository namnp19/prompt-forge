# Fix: File Text UTF-8 Bị Phát Hiện Nhầm Là Binary

## Mô Tả Vấn Đề

Trong `src/utils/file-system.ts`, hàm `analyzeByteContent` đang sử dụng heuristic sai để phát hiện file binary. Cụ thể, dòng code:

```typescript
// Very high bytes that are uncommon in text
else if (byte > 126) {
    nonPrintableCount++
}
```

Hàm này đếm tất cả byte có giá trị > 126 là "ký tự không in được" (non-printable), sau đó nếu > 30% byte là như vậy thì file bị đánh dấu là binary.

**Nguyên nhân lỗi:** UTF-8 mã hóa ký tự non-ASCII (tiếng Việt, Trung Quốc, Nhật, Hàn, Arabic, emoji,...) bằng các multi-byte sequence, trong đó **tất cả các byte đều có giá trị > 127**. Ví dụ:
- Ký tự tiếng Việt "ệ" (U+1EC7) → 3 bytes: `0xE1 0xBB 0x87` (tất cả > 126!)
- Ký tự "中" (U+4E2D) → 3 bytes: `0xE4 0xB8 0xAD` (tất cả > 126!)

Một file văn bản tiếng Việt UTF-8 thường có 50-70% byte > 126, vượt ngưỡng 30% → bị phân loại sai là binary, bị bỏ qua khi tạo prompt LLM.

## Phạm Vi Ảnh Hưởng

Hàm `analyzeByteContent` được gọi qua `looksBinary` và `isBinaryFile` tại hai nơi:
1. **`src/prompts/index.ts`** (line 136): Skip file trong `generateFileContents()` → file văn bản tiếng Việt bị bỏ qua khi build prompt
2. **`src/services/token-counter.ts`** (lines 88, 187): Bỏ qua đếm token → token count = 0

## Giải Pháp

### Thay đổi duy nhất: Sửa hàm `analyzeByteContent` trong `src/utils/file-system.ts`

Thay thế logic byte-level đơn giản bằng cách dùng `TextDecoder` với chế độ `fatal: true`:

**Chiến lược:**
1. Thử decode chunk bằng UTF-8 với `fatal: true`
2. Nếu thành công → file là UTF-8 text hợp lệ → chỉ coi là binary nếu có null byte
3. Nếu throw lỗi (invalid UTF-8) → fallback sang kiểm tra chỉ null bytes + control characters (không tính byte > 126 nữa)

```typescript
function analyzeByteContent(chunk: Uint8Array): boolean {
    if (chunk.length === 0) return false

    // Try to decode as UTF-8 with strict mode.
    // Valid UTF-8 text files (including Vietnamese, Chinese, Japanese, emoji, etc.)
    // will decode successfully. Binary files typically contain invalid UTF-8 sequences.
    try {
        new TextDecoder('utf-8', { fatal: true }).decode(chunk)
        // Successfully decoded as valid UTF-8 text.
        // Only flag as binary if null bytes are present (extremely rare in real text).
        for (const byte of chunk) {
            if (byte === 0) return true
        }
        return false
    } catch {
        // Not valid UTF-8. Fall back to counting only truly suspicious bytes:
        // null bytes and C0 control codes (excluding common whitespace).
        let nullByteCount = 0
        let controlCount = 0

        for (const byte of chunk) {
            if (byte === 0) {
                nullByteCount++
            } else if (byte < 32 && byte !== 9 && byte !== 10 && byte !== 13) {
                controlCount++
            }
        }

        // More than 1% null bytes → binary
        if (nullByteCount > chunk.length * 0.01) {
            return true
        }
        // More than 30% control characters (excl. whitespace) → binary
        if (controlCount > chunk.length * 0.3) {
            return true
        }

        return false
    }
}
```

**Lý do chọn approach này:**
- `TextDecoder('utf-8', { fatal: true })` là Web API chuẩn, có sẵn trong Node.js (v11+) và VS Code extension host
- Xử lý đúng tất cả UTF-8 text files, không phân biệt ngôn ngữ
- File binary thực sự (exe, png, pdf...) đã được bắt trước bởi extension check + magic numbers
- File không phải UTF-8 hợp lệ nhưng là text (Latin-1, Windows-1252) sẽ fail UTF-8 decode, nhưng do không có null bytes và ít control chars nên sẽ KHÔNG bị đánh nhầm là binary → đúng
- Không thay đổi API public, không ảnh hưởng code gọi hàm này

## Rủi Ro

**Thấp:** Một số file binary "exotic" mà tình cờ là valid UTF-8 bytes nhưng không bị bắt bởi magic numbers sẽ không được phát hiện. Tuy nhiên, trong thực tế đây là cực kỳ hiếm vì:
1. Hầu hết binary files đã bị bắt bởi extension check (`BINARY_EXTENSIONS` set)
2. Common binary formats đều bị bắt bởi magic numbers
3. Extension này xử lý source code → không cần lo về binary edge cases
