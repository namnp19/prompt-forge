# Plan: Hiển thị inline message trong PreviewTable khi Apply All thất bại

## Mô tả vấn đề

Khi người dùng làm theo flow:
1. Nhập XML → nhấn **Preview Changes** → PreviewTable hiển thị danh sách các file/patch
2. Nhấn **Apply Changes** (apply all)

→ Nếu một số patch thất bại, **PreviewTable không hiển thị inline message lỗi** cho từng dòng, trong khi flow apply từng dòng riêng lẻ (nút apply ở mỗi row) lại hiển thị được.

## Phân tích nguyên nhân

### Flow Apply Row (đang hoạt động đúng)
- `handleApplyRow(index)` → gửi `applyRowChange` → nhận `applyRowChangeResult`
- Handler cập nhật `rowResults[rowIdx]` với thông tin success/error
- `PreviewTable` nhận `rowResults` và hiển thị inline message cho từng row ✅

### Flow Apply All (bị lỗi)
- `handleApply()` → gửi `applyChanges` → nhận `applyChangesResult`
- Handler hiện tại chỉ cập nhật `results` và `errors` state, **KHÔNG cập nhật `rowResults`**
- `previewData` vẫn còn (không bị clear) → `PreviewTable` được hiển thị
- Nhưng `rowResults` rỗng → không có inline feedback per-row ❌

### Logic hiển thị trong `index.tsx`
```tsx
{previewData ? (
  <PreviewTable ... rowResults={rowResults} />  // rowResults luôn rỗng sau apply all
) : (
  <ResultsDisplay results={results} errors={errors} />  // bị ẩn khi previewData tồn tại
)}
```

### Thêm: Edge case - lỗi parse khi apply all (khi `previewData` đang tồn tại)
- Backend trả về `applyChangesResult` với `success: false`, `errors: [...]`, không có `results`
- Frontend set `errors` nhưng vì `previewData` tồn tại, `ResultsDisplay` bị ẩn → lỗi không hiển thị được

## Giải pháp

Chỉ cần thay đổi **`src/webview-ui/src/components/apply-tab/index.tsx`**:

### Thay đổi 1: Cập nhật `rowResults` khi nhận `applyChangesResult`

Trong handler `applyChangesResult`:
- Khi `message.results` có dữ liệu: map từng result theo index vào `rowResults`
- Điều này cho phép `PreviewTable` hiển thị inline success/error message cho từng row

```typescript
if (message.command === 'applyChangesResult') {
  setIsApplying(false)
  if (message.success) {
    setResults(message.results || [])
    setErrors(null)
    // Map results vào rowResults để hiển thị inline trong PreviewTable
    if (message.results && message.results.length > 0) {
      const newRowResults: Record<number, RowResult> = {}
      for (let i = 0; i < message.results.length; i++) {
        const result = message.results[i]
        newRowResults[i] = {
          success: result.success,
          message: result.message,
        }
      }
      setRowResults(newRowResults)
    }
  } else {
    setErrors(message.errors || ['Unknown error occurred'])
    setResults(null)
    // Clear previewData để ResultsDisplay hiện ra với error message
    // (trường hợp lỗi parse/global - không có results per-row)
    setPreviewData(null)
  }
}
```

### Thay đổi 2: Reset `rowResults` khi bắt đầu Apply All

Trong `handleApply`, thêm `setRowResults({})` để tránh stale data:

```typescript
const handleApply = useCallback(() => {
  // ...
  setIsApplying(true)
  setResults(null)
  setErrors(null)
  setRowResults({})  // <-- thêm dòng này
  onApply(cleaned)
}, [onApply, responseText])
```

## Lý do không cần thay đổi backend

Backend `_handleApplyChanges` trong `index.ts` luôn trả về:
```typescript
{
  command: 'applyChangesResult',
  success: true,       // true ngay cả khi một số file action thất bại
  results: [           // mảng kết quả per-file-action, theo đúng thứ tự
    { path, action, success, message },
    ...
  ]
}
```

Và `success: false` chỉ khi có lỗi parse XML hoặc exception không bắt được. Index của `results[i]` khớp với `previewData.rows[i]` vì cả hai đều xử lý `fileActions` theo thứ tự từ XML parser.
